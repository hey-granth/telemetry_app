Telemetry App Documentation

Production-grade IoT data platform for sensor data ingestion, persistence, aggregation, and realtime streaming.

Project Overview

The Telemetry App is a comprehensive solution designed to handle high-volume sensor data from IoT devices (specifically ESP32). It provides a robust backend for data ingestion and a modern Flutter-based frontend for real-time visualization.

Backend

Built with FastAPI and PostgreSQL. Handles device authentication, high-throughput data ingestion, and provides REST APIs and WebSocket streams.

Learn more →

Frontend

A cross-platform Flutter application. Features real-time charts, device management, and alert configuration. Uses Riverpod for state management.

Setup Guide →

Key Features

  • Real-time Streaming: Live data updates via WebSockets.
  • Scalable Ingestion: Asynchronous processing pipeline.
  • Device Management: Secure provisioning and key management.
  • Historical Data: Efficient storage and querying of time-series data.
Note: This documentation covers version 1.0.0 of the Telemetry App.
